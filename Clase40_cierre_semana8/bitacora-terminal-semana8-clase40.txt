Entorno usado: GitHub Codespaces (Debian Linux)

1. ¿Qué aprendí sobre el sistema operativo como base de una plataforma?

Respuesta:
Aprendí que el sistema operativo es el que permite que los 
programas funcionen y administra los recursos del computador,
 como la memoria, los archivos y los procesos.

2. ¿Qué comandos de terminal usé durante la semana?

Respuesta:
pwd, ls, cd, mkdir, touch, cp, mv, rm, cat, nano, node, npm,
 git, git status, git add, git commit, git push.

3. ¿Qué aprendí sobre usuarios, grupos y permisos?

Respuesta:
Aprendí que los usuarios tienen permisos para acceder a 
archivos y carpetas, y que los grupos ayudan a compartir 
permisos entre varias personas.

4. ¿Qué significa mínimo privilegio?

Respuesta:
Significa que cada usuario o programa debe tener solo los 
permisos necesarios para realizar su trabajo y no más.

5. ¿Por qué un permiso como 777 puede ser riesgoso?

Respuesta:
Porque cualquier usuario puede leer, modificar o eliminar 
el archivo, lo que puede causar problemas de seguridad.

6. ¿Qué aprendí sobre procesos?

Respuesta:
Aprendí que un proceso es un programa que está en ejecución 
y que el sistema operativo se encarga de administrarlo.

7. ¿Qué puede pasar si un proceso se detiene?

Respuesta:
El programa deja de funcionar y la tarea que estaba 
realizando puede quedar incompleta.

8. ¿Qué aprendí sobre logs?

Respuesta:
Aprendí que los logs son registros donde se guarda 
información sobre lo que sucede en el sistema o en una 
aplicación.

9. ¿Cómo ayudan los logs al diagnóstico?

Respuesta:
Ayudan a encontrar errores porque muestran los eventos y 
mensajes que ocurrieron antes de un problema.

10. ¿Qué aprendí sobre Git y trazabilidad?

Respuesta:
Aprendí que Git guarda el historial de cambios de un 
proyecto y permite saber quién hizo cada cambio y cuándo.

11. ¿Qué aprendí sobre archivos .env y tokens?

Respuesta:
Aprendí que el archivo .env sirve para guardar datos 
sensibles, como tokens y claves, sin ponerlos directamente 
en el código.

12. ¿Cómo puede servir Telegram a una comunidad digital?

Respuesta:
Puede servir para enviar avisos, compartir información y 
recibir notificaciones de forma rápida.

13. ¿Qué cuidados debo tener antes de integrar Telegram con Node.js o n8n?

Respuesta:
No compartir el token del bot, guardar las credenciales en 
el archivo .env y probar que la configuración funcione 
correctamente.

14. ¿Qué fue lo más importante de la Semana 8?

Respuesta:
Aprender a usar la terminal, entender mejor Linux, 
trabajar con Git y conocer cómo conectar un bot de Telegram 
usando Node.js.
